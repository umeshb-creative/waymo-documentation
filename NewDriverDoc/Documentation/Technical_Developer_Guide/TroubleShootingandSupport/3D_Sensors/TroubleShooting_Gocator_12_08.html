Troubleshooting Guide for Gocator 2530 and 3506

This guide provides comprehensive solutions for diagnosing and resolving common issues related to hardware, desktop applications, connectivity, network configurations, and driver installations associated with Gocator 2530 and 3506. By following the troubleshooting steps outlined, users can effectively address and rectify challenges encountered during the setup and operation of these 3D Lasers.

Trouble Shooting Connection and Laser Profiling

Issue Possible Cause Solution
Gocator Sensor not detected When connecting with a web browser, the sensor is not found (page does not load). 1.Verify the sensor power is on. This will be indicated by an illuminated POWER LED.
2.Verify the Power & Ethernet cordset is connected to the Power/LAN connector and the Ethernet RJ45 plug is connected to the Ethernet switch.
3.Verify that the client computer’s network settings are properly configured. See Getting Started > Network Setup in the Gocator user manual or your computer’s documentation on configuring a network adapter.
4.Download the Gocator Tools package from the downloads area of LMI’s website at www.lmi3D.com. Unzip and run the Sensor Discovery Tool [Discovery>kDiscovery.exe] to verify that the sensor has the correct network settings.
Not able to login When attempting to log in, the password is not accepted. Download the Gocator Tools package from from the downloads area of LMI’s website at www.lmi3D.com. Unzip and run the Sensor Discovery Tool [Discovery>kDiscovery.exe] to find the sensor on the network and click Factory Restore.
WARNING: Performing a factory restore resets configuration settings to their original values. These settings can be recovered from backup files if you have previously saved them.
When the Play button is pressed, the sensor does not emit laser light. Power Failure / LAN Connector Issue 1.Ensure that the decal covering the laser emitter window, normally affixed to new sensors, has been removed.
2.Verify that the LASER LED on the Gocator is illuminated. If not, the laser safety input signal is off. To determine the correct solution for your application, see Specifications > Gocator Power/LAN Connector > Laser Safety Input in the user manual.
3.The exposure setting may be too low. For more information, see Gocator Web Interface > Scan Setup and Alignment > Sensor > Exposure in the user manual.
The sensor emits laser light, but the Range Indicator does not illuminate and/or points are not displayed in the data viewer Measurement Target is out of range or Exposure Time is not reasonable 1.Verify that the measurement target is within the sensor’s field of view and measurement range. The RANGE LED on the Gocator will illuminate when the target is in range.
2.Check that the exposure time is set to a reasonable level. For more information, see Gocator Web Interface > Scan Setup and Alignment > Sensor > Exposure in the user manual.
The sensor CPU level is near 100%. Memory Full or High Trigger Speed/Profiling Resolution 1.Review the active measurements and remove any that are unnecessary.
2.Consider reducing the trigger speed.
3.Consider reducing the laser profiling resolution.

Gocator Not Connecting Problem with Calling OpenDevice API 1.Verify Gocator connection using GoSDK.
2.Ensure the connection is closed before calling OpenDevice API through gRPC.
3.Refer to Error Codes and Descriptions for further troubleshooting.
