Troubleshooting Guide for Keyence CL3000

​This guide provides comprehensive solutions for diagnosing and resolving common issues related to hardware, desktop applications, connectivity, network configurations, and driver installations associated with Keyence CL3000. By following the troubleshooting steps outlined, users can effectively address and rectify challenges encountered during the setup and operation of these controllers.

Hardware-Trouble Shooting Guide

Troubleshooting Hardware Issues

Issue Possible Cause Solution
The PWR/ERR LED do not light after switching ON the controller. 1.Power cable not connected properly
2.Power supply not meeting the specification range		 1.Connect the power cable correctly
2.Be sure to use a power supply that meets the specification range.
The PWR/ERR LED of the controller is lit (Red). An error is displayed on the screen. Error message appear when using CL-NavigatorN is used for the communication. Take the necessary measures by following the displayed error information.
The HEAD LED does not light. Head cable is not firmly connected to the fiber adapter. Head cable or head extension cable is broken. 1.Plug the head cable firmly into the fiber adapter.
2.Try again with a different head or head cable
No light is being emitted. 1.System may be in setting mode
2.If Encoder pulse signal is coming in when the operation mode of the encoder unit is set to “TRIGGER”
3. Emission stop input may be ON		 1.The sensor does not emit while in setting mode. Exit setting mode and switch to measurement mode. setting mode, and move to the measurement mode.
2.Enter the encoder pulse signal correctly.
3.Set the emission stop input to OFF
The measurement value becomes -FFFFFF. Light is being emitted from the sensor head. -Turn OFF emission stop if it is ON.
-Check that the encoder pulse signal is entered correctly
The measurement value becomes -FFFFFF. Target object may be inside the measurement range. Place the target inside the measurement range
The measurement value becomes -FFFFFF. Displacement (Transparent object]) or Thickness (Transparent object)] as the measurement method Selected [Target peak] is specified when performing the measurement described on the left, but the measurement value will be -FFFFFF if the specified peak does not exist. (For example, If +1P and +2P (Two reflection lights) are specified in [Thickness (Transparent object)] but the target is non-transparent and only +1P (One reflection light) exists, the measurement value will be -FFFFFF.)
The measurement value becomes -FFFFFF. Check the minimum display unit, offset, and scaling settings. For example, when the minimum display unit is “0.00001 mm”, “±9.99999mm” will be the upper and lower limits of the measurements that can be displayed. When the measurement value exceeds these limits, “+ FFFFFF” or “-FFFFFF” will be displayed (Display range exceeded). Change the minimum display unit or adjust the offset/ scaling settings, such that the measurement value will be in a displayable range.
The measurement value becomes -FFFFFF. Check the waveform data. Mask setting may be enabled. Clear the mask setting, or move the target to a position outside the masked area.
The measurement value becomes -FFFFFF. Check the waveform data. Peak may be too high and saturated. 1.If the light quantity control is set to [MANUAL], change it to [AUTO].
2.Check if some of the reflection light intensity is being blocked by an obstacle. --Try setting the detection threshold to [AUTO], or changing it to a smaller number in [MANUAL].
4.Try making the sampling cycle longer.
5.The calibration of light intensity may have been performed incorrectly. Perform a calibration of light intensity again, or reset the calibration of light intensity to the factory default settings.
The measurement value becomes -FFFFFF. 1.Light is being emitted from the sensor head.
2.Target object may be inside the measurement range.
3.Check the waveform data. Mask setting may be enabled.
4.Check the waveform data. Peak may be too low.
The measurement value is displayed as “++++++”. 1.Stop measurement input may be ON
2.Light off input may be turn ON
3.Hold mode settings
4.Measurement value is displayed after waiting for a while		 1.Set the stop measurement input to OFF
2.Turn OFF the light off input.
3.If set to other than [Normal], [Auto Peak hold] or [Auto bottom hold], the measurement value is finalized according to the external timing or internal timing. Change the hold processing or follow the timing process correctly.
4.If the averaging times is high, time will be needed for “Sampling cycle x Averaging “ before the measurement value can be finalized. The smaller the averaging times, the shorter it will take to display the measurement values.
The control input is not taking effect correctly. Input cable is not wired correctly Check in [I/O Monitor]. Re-wire the input terminals that do not change correctly.
The judgment output is not output correctly. Output cable is not wired correctly Check in [I/O Monitor]. Re-wire the output terminals that do not change correctly.
Nothing appears in the display panel. Display panel cable is not connected correctly or broken Connect the display panel cable correctly or Try using another display panel cable.
Button operation does not work on the display panel. Panel lock may be ON. Switch the panel lock to OFF.
The PC freezes while using CLNavigatorN. System environment of the PC may not be meeting the specification range Be sure to use it under a system environment that meets the specification range.
Ethernet connection to the controller fails when CLNavigatorN is started. Ethernet is not selected in the [PC Connection Settings] Select Ethernet in [PC Connection Settings] and set the IP address correctly.
Ethernet communication with external devices does not work. IP address is not set correctly Connect via USB to check the IP address setting of the controller in [Environment settings], and then configure the settings correctly.

CL3000 Not Connecting Problem with Calling OpenDevice API 1.Ensure the CL3000 Connection using CL-Navigator N.
