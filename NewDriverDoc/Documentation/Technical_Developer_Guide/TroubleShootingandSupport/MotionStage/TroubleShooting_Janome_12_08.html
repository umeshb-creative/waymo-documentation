Troubleshooting Guide for Janome JR3304/JR3303

​This guide provides comprehensive solutions for diagnosing and resolving common issues related to hardware, desktop applications, connectivity, network configurations, and driver installations associated with Janome JR3304/JR3303. By following the troubleshooting steps outlined, users can effectively address and rectify challenges encountered during the setup and operation of these controllers.

Hardware-Trouble Shooting Guide

Troubleshooting guide for power and connectivity Issues:

The LED indicators on the Janome JR3203N-AC robot provide important status information regarding operation, errors, and start inhibition conditions. Below are the possible LED statuses, their meanings, associated issues, and troubleshooting steps.

Issue Possible Cause Solution
Red–Blinking: Start to Inhibit 1.Start inhibition signal is ON (#sysIn2, #sysIn14)
2.Robot is in a stopped state (#sysOut2 ON)
3.I/O-S Interlock is set		 1.Check and disable the start inhibition signal.
2.Ensure the robot is not in a stopped state.
3.Verify I/O-S function settings in TP Menu.
Green - Blinking The robot is waiting for an external start signal Send a Start instruction from the start channel
Green - Slow Blinking External Start signal detected, but execution has not begun Ensure the start channel signal is active and correct
Green - Solid (Lit) The robot is operating normally No action needed
Red & Green - Alternate Blinking An emergency stop is in progress 1.Check the emergency stop button and reset if necessary
2.Verify safety circuits and restart

Troubleshooting Hardware & Connection Issues

Issue Possible Cause TroubleShooting Steps
Janome Robot not Recognized in Windows Device Manager. Driver not installed. Install or update JR C-Points II from the official website.
Robot Stops Unexpectedly During Operation Overheating or low supply voltage.A Sudden interruption in the power supply. Internal system fault. Check for error codes on the display or logs. Ensure a stable power supply is connected. Restart the robot and retry.
Emergency Stop Switch Does Not Release 1.A mechanical issue with the Emergency Stop switch.
2.The switch is not turned correctly to release mode.		 1.Press down firmly and turn the switch clockwise to release.
2.If the switch remains stuck, check for physical damage or replace it.

Troubleshooting Desktop Application(JR C-Points)Related Issues:

Issue Category Possible Cause
Difficulty installing JR C-Points II on a PC 1.Ensure administrator account rights during installation.
2.If JR C-Points II Limited Edition is already installed, uninstall it before proceeding with the new installation
Driver Installation 1.Verify that the JR C-Points II and drivers are installed and up to date. Check for driver conflicts in Device Manager (Windows).
2.Reinstall drivers and restart the system
Connection Problems Between PC and Robot: The PC fails to connect with the robot via Ethernet. 1.Confirm the Ethernet cable is securely connected to both the PC and the robot's Ethernet port.
2.Disconnect the teaching pendant before using JR C-Points II, as the robot doesn't support simultaneous connections
3.Ethernet Settings Configuration:
  • In JR C-Points II, navigate to [Robot] > [Ethernet Settings].
  • Input the correct IP address and port settings matching the robot's configuration.
Commands issued from JR C-Points II don't execute on the robot. 1.Confirm the robot is in the correct mode to receive commands from the PC.
2.Ensure the emergency stop isn't engaged. If it is, release it and reinitialize the robot.
3.After powering on or releasing an emergency stop, perform mechanical initialization:
  • In JR C-Points II, go to [Robot] > [Initialize].
  • Follow on-screen prompts to complete the process.

Troubleshooting Driver Related Issues

Issues Possible Cause Troubleshooting Steps
Janome Not Connecting Problem with Calling OpenDevice API 1.Ensure the Janome robot is responding via ping command
2.Close any existing connections before calling the OpenDevice API via gRPC.
3.Check for timeout errors or specific error codes returned by OpenDevice.
4.Always close the connection properly before reattempting OpenDevice via gRPC.
5.Make sure right C&T file is uploaded
Mechanical Initialization Does Not Start 1.Incorrect Start Channel selected.
2.Incorrect command/signal format from the external device.
3.Emergency Stop is active, preventing initialization. 		1.Release the Emergency Stop before attempting initialization.
2.Verify the Start Channel is correctly configured:TP PC MODE → Administration → Administration Settings Mode → Start Channel → Robot.
3.If initializing via Ethernet, send the command "Initialize R0" from an external device (e.g., PC).
4.Run the CLI command: MechaInitialization or perform Mechanical Initialization from JR-C Points Desktop Application
5.Confirm in JR C-Points II Software that the robot is in external RUN mode.
The Robot Does Not Restart After Releasing the Emergency Stop 1.Mechanical Initialization was not executed after E-Stop release.
2.A safety fault is detected, preventing restart.		 1.Perform Mechanical Initialization using the correct Start Channel (Ethernet).
2.Check for safety system faults and reset them if necessary
Unable to Select the program Problem with Calling SelectProgram API Verify if the program exists on the controller by using the ListProgram API.
Program not running Problem with Calling RunProgram API 1.Ensure a valid program is selected before running.
2.If the controller has undergone a power cycle, follow these steps:
  • Perform Mechanical Initialization.
  • Select the program using SelectProgram.
  • Run the program using RunProgram.

3.Verify if another program is already running, which may prevent execution.
PTP Movements not working AbsolutePtpMovementControl and RelativePtpMovementControl 1.If the controller has undergone a power cycle, Perform Mechanical Initialization.
2.Check for non-zero velocity and acceleration
3.Verify if the robo is in External Mode
4.Check if any errors are there
5.Do not try running motion APIs when robot is busy or in a cycle and software is busy or performing other functions
Unable to Add positions in program Problem with Calling AddPosition API 1.Verify is the program exists to which positions needs to be added
2.To add a position to a program, the position to be added should be less than/ equal/one greater than the current position
Unable to Add a program using AddPositon Problem with Calling AddPosition API 1.Verify is the program exists to which positions needs to be added
2.To add a position to a program, the position to be added should be less than/ equal/one greater than the current position
Unable to Add positions in program Problem with Calling AddPosition API Verify the sequential order of the program (For Ex, a position cannot be added
Unable to Set IO Problem with Calling ReturnHome API If the controller has undergone a power cycle, Perform Mechanical Initialization.
