Troubleshooting Guide for Animatic Motor (sm23165d-cds7)

This guide provides comprehensive solutions for diagnosing and resolving common issues related to hardware, desktop applications, connectivity, network configurations, and driver installations associated with Animatic Motor(sm23165d-cds7). By following the troubleshooting steps outlined, users can effectively address and rectify challenges encountered during the setup and operation of these smart motors.

Hardware-Trouble Shooting Guide

Understanding LED Indicators and Troubleshooting Power Issues:

​The Animatic SmartMotor features three status LEDs (LED0, LED1, and LED2) that provide insights into the motor's operational state and assist in diagnosing issues. Below is a table detailing common LED indications, their possible causes, and recommended troubleshooting steps:​

Issues Possible Cause Troubleshooting Steps
LED 0 is Off (No Power) The power adapter is not connected or is faulty. 1.Ensure the power adapter is properly plugged in and working.
2.Ensure proper voltage is provided.
3.Try a different adapter.
LED 0 is Solid Green (Drive On, but motor not moving). The Motor is enabled but no trajectory command is given. Send a motion command via SMI software or controller.
LED 0 is Flashing Green (Drive Off) Motor is disabled Enable the motor using SMI software or the appropriate API command.
LED 0 is Flashing Red (Watchdog Fault). The system watchdog detected an error. Power cycle the motor and controller. If issue persists, check system logs for errors.
LED 0 is Solid Red (Major Fault). Overcurrent, overheating, or other serious fault. Check motor temperature, power connections, and controller fault logs.
LED 0 Alternating Red/Green (Boot Load Mode). Motor firmware needs to update or is corrupt. Reflash the firmware using manufacturer tools.
LED 1 is Off (Not Busy, No Motion). No trajectory command given. Send a motion command using SMI software or API.
LED 1 is Solid Green (Trajectory in Progress but no movement). Motion parameters might be incorrect. Verify target position, speed, and acceleration settings.
LED 2 (CAN Bus Fault Red LED) - Single Flash. At least one error exceeded limit. Check CAN wiring, baud rate, and ensure all devices on the network are properly configured.
LED 2 (CAN Bus Fault Red LED) - Double Flash. Heartbeat or guard error Check if the CANopen master is running and correctly monitoring the motor.
LED 2 (CAN Bus Fault Red LED) - Solid. CAN bus communication is off Ensure correct termination resistors and check network integrity.
LED 3 (CAN Bus Status Green LED) - Blinking. Pre-operational state (Boot-up) Wait for the system to finish initialization. If stuck, check firmware and CAN settings.
LED 3 (CAN Bus Status Green LED) - Solid. Normal operation No action is required, the system is running correctly.
LED 3 (CAN Bus Status Green LED) - Single Flash. The Device is in stop state. Restart the device or send a start command.

Note:For Class 5 D-style AD1 option motors, at no time should the voltage to any input exceed the level on the I/O power input (Pin 11). Doing so could cause immediate damage to the expanded I/O hardware.

Troubleshooting Port and Connection Issues

Issues Possible Cause Troubleshooting Steps
The motor control power light does not illuminate. 1.Control power is off, disconnected, or incorrectly wired.
2.The motor has routed drive power through drive enable pins.
3.The motor is equipped with the DE option.		 1.Check that the control power is connected to the proper pins and turned on.
2.Ensure cabling is correct and drive power is not being delivered through the 15-pin connector.
3.To energize control power, apply 24-48 VDC to pin 15 and ground to pin 14.
The motor does not communicate with SMI. 1.Transmit, receive or ground pins are not connected correctly.
2.The motor program is stuck in a continuous loop or is disabling communications.		 1.Ensure that transmit, receive and ground are all connected properly to the host PC.
2.To prevent the program from running on power up, use the Communications Lockup Wizard located and clear EEPROM on the SMI software Communications menu. If stuck in bootloader, contact Moog Animatics
Motor disconnects from SMI sporadically. 1.COM port buffer settings are too high.
2.Poor connection on the serial cable.
3.Power supply unit (PSU) brownout.		 1.Adjust the COM port buffer settings to their lowest values.
2.Check the serial cable connections and/or replace it.
3.PSU may be too high-precision and/or undersized for the application, which causes it to brown out during motion. Make moves less aggressive, increase PSU size or change to a linear unregulated power supply.
The motor stops communicating after power reset, and requires re-detection. The motor does not have its address set in the user program.
NOTE: Serial addresses are lost when motor power is off or reset.		 Use the SADDR or ADDR= command within the program to set the motor address.
Red PWR SERVO light illuminated. Critical fault. To discover the source of the fault, use the Motor View tool located on the SMI software Tools menu.

Troubleshooting Common Faults:

Issues Possible Cause Troubleshooting Steps
Bus voltage fault. Bus voltage is either too high or too low for operation. Check servo bus voltage. If the motor uses the DE power option, ensure that both drive and control power are connected.
Overcurrent occurred. The motor intermittently drew more than its rated level of current. Does not cease motion. Consider making motion less abrupt with softer tuning parameters or acceleration profiles.
Excessive temperature fault. The motor has exceeded the temperature limit of 85°C. The motor will remain unresponsive until it cools down below 80°C. The motor may be undersized or the ambient temperature is too high. Consider adding heat sinks or forced air cooling to the system.
Excessive position error. The motor's commanded position and actual position differ by more than the user-supplied error limit. Increase error limit, decrease load or make movement less aggressive.
Historical positive/negative hardware limit faults. 1.A limit switch was tripped in the past
2.The motor does not have limit switches attached.		 1.Clear errors with the ZS command.
2.Configure the motor to be used without limit switches by setting their inputs as general use.

Troubleshooting Desktop Application(SMI Interface)-Issues

Issues Possible Cause Troubleshooting Steps
SMI Software Not Detecting Animatic Motor The Computer not configured properly for RS-232 communication 1.Restart the computer before launching SMI.
2.Ensure SmartMotor is powered on.
3.Check the COM port settings in SMI and manually select the correct port.
4.Try using a different computer if the issue persists.
No Communication with Smart Motor. Incorrect cable connection or damaged cable. 1. Verify the RS-232 connection using SMI Interface
2.Try a different serial cable and port.
3.Ensure the correct baud rate and communication settings in SMI.
Motor Not Powering On No power supply or faulty connection. 1.Check the power supply voltage (match specifications).
2.Inspect power cable connections.
3.Try a different power source to rule out power issues.
Motor Not Moving After Detection. Motor disabled or in an error state 1.Use SMI to check for error messages or faults.
2.Issue the EN (Enable Motor) command.
3.Reset and power cycle the motor.
Unresponsive Motor After Command Execution Corrupt firmware or incorrect settings 1.Perform a factory reset of the SmartMotor.
2.Reflash the firmware using SMI.
3.Ensure the command format is correct in SMI.
RS-232 / RS-485 Not Responding The wrong COM port was selected in the software 1.. Open Device Manager (Windows) and find the active COM port.
2.Match the COM port in SMI settings.
3.Try different baud rate settings (e.g., 9600, 115200).
Positioning Error / Loss of Steps Encoder feedback issues or misalignment 1.Check encoder feedback wiring.
2.Verify the encoder resolution settings.
3.Recalibrate motor positioning.

Note: For D-style SmartMotors, pins 14 and 15 of the 15-pin D-sub port are intended for use with DE series motors for control power only.
Attempts to power a non-DE motor through those pins as main servo drive power could result in immediate damage to the electronics and will void the warranty.
Do not use the Address Motors feature. Doing so will overwrite the existing motor addresses.

Troubleshooting Driver Related Issues

Issues Possible Cause Troubleshooting Steps
Animatic Smart Motor Not Connecting Problem with Calling OpenDevice API 1.Ensure the Animatic Motor is properly connected using SMI Interface.
2.Close any existing connections before calling the OpenDevice API via gRPC.
3.Check for timeout errors or specific error codes returned by OpenDevice API.
4.Always close the connection properly before reattempting OpenDevice via gRPC.
Several commands not recognized during compiling. Compiler default firmware version set incorrectly. Use the Compiler default firmware version option in the SMI software Compile menu to select a default firmware version closest to the motor's firmware version. In the SMI software, view the motor's firmware version by right-clicking the motor and selecting Properties.
Motor not connecting 1.Wrong port number or incorrect motor selected
2.Faulty or loose cable connection		 1.Ensure the correct COM port number is selected in SMI software. Verify that the correct motor ID is being accessed.
2.Inspect all communication cables (RS-232) and reconnect if needed.
The Motor Not Moving The Limit Switch has been Set (Positive or Negative limit switch). Clear All Flags and Clear Fault Bits by calling Enable Motor and Home the Motor using Return Home API.
The motor moves but the direction is reversed Incorrect direction configuration Provide the correct directions and velocity
