Troubleshooting Guide for Banner SC10- Safety PLC

This guide helps diagnose and resolve common hardware, desktop application, and driver issues with the Banner SC10- Safety PLC

Hardware-Trouble Shooting Guide

Understanding LED Indicators and Troubleshooting Power Issues:

Issue Possible Cause Solution
All LEDs Off 1.Safety Controller is powered off.
2.Initialization mode active.		 1.Ensure 24V DC power is properly connected.
2.Wait for the initialization sequence to complete.
Power/Fault LED (Green, Flashing) Configuration or Manual Power-Up mode active. If using SC-XM3 for configuration, cycle power to apply changes.
Power/Fault LED (Red, Flashing) Non-operating Lockout condition Check for safety lockout conditions and resolve them before restarting
Inputs (Green, Solid) 24V DC power is supplied with no faults Set the camera to a Static IP using Pylon IP Configurator.
Inputs (Red, Solid)/b> 1.0V DC is supplied with no faults
2.Input configured as status output and inactive 		1.Check if the correct voltage is applied to the input.
2. Check if the correct voltage is applied to the input.
RO1, RO2 Relay Outputs (Green, Solid) Contacts closed (relay output is on) No action needed (normal operation)
RO1, RO2 Relay Outputs (Red, Solid) 1.Contacts open (relay output is off)
2.Output not configured 		Verify relay output configuration in the Safety Controller settings
RO1, RO2 Relay Outputs (Red, Flashing) 1.Safety Output fault detected
2.EDM (External Device Monitoring) fault detected
3.AVM (Automatic Verification Monitoring) fault detected 		Inspect connected devices for faults Check EDM and AVM settings for errors
Ethernet Diagnostic LEDs (Amber LED On, Green LED varies with traffic) Normal operation No action needed
Ethernet LEDs (Both Off) Hardware failure 1.Inspect network cables and ensure proper connectivity
2.Restart the controller and check again
Ethernet LEDs (5 Flashes Followed by Several Rapid Flashes) Normal power-up sequence No action needed (expected behaviour)
Ethernet LEDs (1 Flash Every 3 Seconds) Possible hardware issue Contact Banner Engineering for support
Ethernet LEDs (2 Flash Repeating Sequence) A cable was unplugged while active in the last 60 seconds Check Ethernet cable connections and reconnect if needed
Ethernet LEDs (3 Flash Repeating Sequence) A cable is unplugged Reconnect the cable and check for proper network function
Ethernet LEDs (4 Flash Repeating Sequence) The network is not enabled in the configuration Enable network settings in the Safety Controller configuration
PROFINET Flash Command (All LEDs Flashing Twice per Second for 4s) "Flash LED" command received from PROFINET network No action needed (normal response to PROFINET flash command)

Troubleshooting Connection and Network Issues :

Issue Possible Cause Solution
SC10 not Recognized in Windows Device Manager. Driver not installed. Install or update the Driver from Here
Network Connection Lost No GigE connection. Connect SC10 directly to the PC to rule out switch issues. Check for firewall restrictions.
SC10 Not Accessible via IP. IP conflict or wrong subnet. 1.Use ipconfig or ping to verify the assigned IP address.
2.Ensure the device and PC are on the same subnet.

Troubleshooting Desktop Application Related Issues:

Issue Solution
Live Mode button is unavailable (greyed out) 1.Make sure the SC-Ethernet cable is plugged into both the computer and the Safety Controller.
2.Verify that the Safety controller is installed properly
3.Exit the Software
4.Unplug the Safety controller and plug it back in.
5.Start the Software
Unable to read from the Safety Controller or send the configuration to the Safety Controller (buttons greyed out) 1.Make sure Live Mode is disabled
2.Verify that the Safety Controller is installed properly
3.Exit the SoftwareClean the lens carefully with a microfiber cloth. Remove the lens cap before use.
4. Unplug the Safety Controller and plug it back in.
5.Start the Software
