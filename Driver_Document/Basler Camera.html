Basler Camera driver document

Introduction

Brief overview of the driver

This project focuses on developing drivers for Basler Camera (ace2 a2A1920-51gcBAS) to be utilized in automation applications, with a particular emphasis on integrating them with a user interface (UI) application for future use. The project aims to create software components that enable seamless communication between Basler Camera and the UI application, allowing for the camera settings, capture of images, and streaming of data. Through this integration, the project seeks to enhance the usability and functionality of Basler Camera within automated processes, contributing to increased efficiency and productivity in various industrial settings.

Purpose of the document

The purpose of this document is to provide a detailed guide for developing a driver for the Basler Camera (ace2 a2A1920-51gcBAS). The driver will enable communication between the Camera and external software application(pylon viewer), allowing to capture an Image, Live Streaming change camera settings like pixel format, exposure time, gain and many more. It covers the software development process, including setting up the development environment, implementing the driver, testing and debugging, and integrating the driver with the Basler Camera.

The primary objective of the document is to provide a comprehensive guide for developed drivers for Basler Camera (ace2 a2A1920-51gcBAS) used in automation using a UI application. This includes detailed instructions, software installation guide, a device description with its features, and best practices to ensure the successful development and integration of drivers with the devices.

Scope and objectives

  1. Driver Development: The document focuses on the development of a driver for Basler Camera using industry-standard protocols.
  2. Technical Information: It includes technical specifications of the device, hardware and software requirements, and information about the Software Development Kit (SDK) provided for the device.
  3. SDK Overview: Introduction to Pylon Viewer and its Installation.
  4. Driver Implementation: It covers the development process, including the choice of programming language, information on APIs, how to use APIs, limitations of the driver, precautions to take before execution, and the actual implementation of the driver.
  5. Testing and Debugging: The scope extends to testing and debugging procedures to ensure the reliability and functionality of the driver for real-time application
  6. Documentation and Support: : Lastly, the document includes information on documentation standards and support resources available for developers.

Overall, the aim is to facilitate the successful development and implementation of the Basler Camera, ensuring it meets the required standards and serves its intended purpose effectively.

Software

Basler pylon Camera Software Suite (version 6.0 or higher)

Available for Windows, Linux x86, Linux ARM, macOS, and Android

Software Overview and Requirements – Basler Pylon Viewer

The pylon Camera Software Suite is a collection of drivers and tools for operating any Basler camera with a Windows, macOS, Android or Linux PC or embedded processing board. As it is based on GenICam technology, it offers barrier-free access to the newest camera models and the latest features. Making changes to an existing camera device in your application essentially becomes a plug and play process.

Features of Basler Camera Pylon

  • The pylon Viewer is a versatile application for testing and evaluating Basler cameras.
  • The tree structure of the viewer’s graphical user interface lets you easily find the best camera parameter setup, adjust image quality, and control advanced camera features.
  • Use predefined use cases to easily find any related parameters for your application use case, e.g. check whether the application is losing image data. The pylon Viewer and APIs allow for camera images to be recorded to video files (MP4 or AVI) or into a sequence of single images.
  • The new Data Processing C++ API extends the existing, powerful C++ API for camera configuration and image acquisition, and allows for rapid application development.

Installation

To install the Basler pylon Camera Software Suite for Windows:

  1. Download the pylon installer from the here.
  2. Launch the downloaded installer.
  3. Follow the instructions in the setup wizard. The wizard will guide you through the installation process. During installation, you can choose the interfaces of your cameras (e.g., USB or GigE) so that only the software you need is installed.

Introduction to Software Development Kit

The pylon Camera Software Suite also contains a powerful SDK and helps to develop any kind of camera application for Windows, Linux, macOS or Android on desktop as well as on embedded systems.

With the well-designed and user-friendly API you will need just a few lines of code to configure the camera and to grab and process images. The Instant Camera Class takes care of device lifetime and buffer management, as well as setting up a grab loop and handling camera events.

Documentation and Samples Comprehensive documentation, plus a collection of programming samples for C ++, C, .NET languages (C#, VB.NET, etc.), illustrate various use cases for different grab strategies, multi-camera applications, recording video files, apply image processing, and many more.

gRPC API’s Description for Basler Camera

  1. OpenDevice

    Description for OpenDevice API: Establishes a connection to the Basler Camera, initializing communication for further operations.

    RPC Command :rpc OpenDevice(.generic.CommandIPRequest) returns(.generic.GenericDeviceModel) {}

    Request:
    message CommandIPRequest{
    string ip_address = 1;
    string com = 2;
    int64 port_address = 3;
    }
    Parameter Description
    ip_address Indicate the IP address of the device
    com COM Port Address of the Device with serial communication (ex. COM1)
    port_address Port address of the device with serial communication (ex. 1)
    Response
    message CommandResponse {
    int32 resp_code = 1;
    string resp_message = 2;
    }
    Parameter Description
    resp_code Response code from the device API
    resp_message Response Message
    CLI Command: python baslercamera.py OpenDevice 192.168.4.20 Basler_1
    Description for CLI Command
    • OpenDevice: API called to the Basler Camera through CLI Command to connect and initialize all resources
    • 192.168.4.20: IP Address of the device
    • Basler_1: Aliased Name for Basler Camera

  2. CloseDevice

  3. GetDeviceDetails

    Description for GetDeviceDetails API: Retrieves detailed information about the connected Basler Camera, including its configuration and status.

    RPC Command : rpc GetDeviceDetails(.generic. CommandRequest) returns (.generic.GenericDeviceModel) {}

    Request:
    message CommandRequest {
    string device_alias = 1;
    }
    Parameter Description
    device_alias Unique alias of the device
    Response:
    message GenericDeviceModel{
    string instance_id = 1;
    string device_alias = 2;
    DeviceType device_type = 3;
    DeviceModel device_model_name = 4;
    optional DeviceStatus device_status = 5;
    string ipaddr = 6;
    string mac_addr = 7;
    google.protobuf.Value conn_handle=8;
    string port_name = 9;
    int64 port_address = 10;
    CommandResponse cmd_response = 11;
    }
    Parameter Description
    instance_id New instance for the device in the enumeration
    device_alias The unique device_alias used to refer the device
    device_type Indicates the type of device like Motion stage, Safety Controller etc (defined below in the DeviceType enum)
    device_model_name Indicate the model name for each device (define below in the DeviceModel enum)
    device_status Indicate whether the device is online , offline or not reachable (define below in the device status enum)
    ipaddr Indicate the ipaddress of the device
    mac_addr Indicate the mac address of the device
    Conn_handle Handle object to keep the device open
    port_name Port name
    port_address Port address
    cmd_response Indicates command response that describes API response code and its description (Defined below in CommandResponse)
    enum DeviceType{
    NONE = 0;
    SENSOR_3D = 1;
    MOTION_STAGE = 2;
    CAMERA = 3;
    IO_MODULE = 4;
    SAFETY_CONTROLLER = 5;
    ANIMATIC_MOTOR=6;
    }
    enum DeviceModel{
    DUMMY_MODEL = 0;
    ED527 = 1;
    ED588 = 2 ;
    JR3000 = 3;
    OPTOE = 4;
    BASLER = 5;
    SPiiPlusEC=6;
    GT6600 = 7;
    MOTOR = 8;
    }
    enum DeviceStatus{
    OFFLINE = 0;
    ONLINE = 1;
    NOT_REACHABLE =2;
    }
    message CommandResponse{
    int32 resp_code = 1;
    string resp_message = 2;
    }
    Parameter Description
    resp_code Response code from the device API
    resp_message Response Message
    CLI Command: python baslercamera.py GetDeviceDetails Basler_1
    Description for CLI Command:
    • GetDeviceDetails: API called to the Basler Camera through CLI Command
    • Basler_1: Aliased Name for device

Limitations of the driver

  1. Compatibility: The driver may only be compatible with ace2 a2A1920-51gcBAS versions of Basler Camera, limiting its use with other devices.
  2. Integration: Integrating the driver with other systems or software may pose challenges, especially if there are compatibility issues or dependencies.
  3. Modifying Environment(Pylon Viewer) may affect Live Streaming.

Precautions to take before executing the driver

Ensure to load good configuration file through LoadConfig API to overcome any issues with driver.

GLOSSARY

Glossary of Terms for Basler Camera:

  • Driver: A software component that enables communication between the operating system and a hardware device, in this case, a camera.
  • SDK (Software Development Kit): A set of tools, libraries, and documentation that developers use to create software applications for a specific platform or framework.
  • API (Application Programming Interface): A set of rules and protocols that allow different software applications to communicate with each other.
  • Firmware: Software that is embedded into a hardware device, such as a camera, to control its operation.
  • Resolution: The number of pixels in an image, typically measured in megapixels (MP), that determines the level of detail and clarity in the image.
  • Frame Rate: The number of frames or images displayed or processed per second, often measured in frames per second (fps).
  • Gain: Gain in the context of a camera refers to the amplification of the signal from the camera sensor. However, increasing the gain too much can introduce noise into the image, reducing its quality. Gain is usually measured in decibels (dB).
  • Exposure Time: : The amount of light that reaches the camera sensor, controlled by settings such as shutter speed, and aperture.
  • Pixel: The smallest unit of a digital image, representing a single point of color.
  • Buffer: A temporary storage area in memory used to hold data, such as images, before it is processed or transferred.
  • Streaming: The process of continuously transmitting data, such as video, over a network in real-time.
