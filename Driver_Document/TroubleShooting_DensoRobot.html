Troubleshooting Guide for Denso 6-axis RC7 Controller

This guide helps diagnose and resolve common hardware, desktop application, and driver issues with the Denso 6-axis RC7 Controller

Follow the steps below to check for power, connection, overheating, and network-related problems.

Troubleshooting Connection with RobotController

TroubleShooting on RC7 Controller

Check Action
Is the NetwoRC ROM version higher than V2.330? If the version is below v.2330, please update the controller ROM Image to v2.330 or above.
Is the cable, RS232C, or Ethernet cable connected properly?
 Ensure that the cable is not loose on the connector.
Is the type of cable such as Straight and Cross correct? Check the cable.
In the case of Ethernet, is the address correctly set? Check the address setting.
In the case of the ethernet connection over the segment, is the default gateway correctly set? Make sure that the Controller IP address in the COM settings is the same between the application and the controller. Make sure that the IP address of the client PC and the IP address of the ext. run on the controller are the same..
Is the communication permission on the communication menu correctly set? Check the communication permission.
Is the ORiN extension set? Add the ORiN option.

Trouble in accessing variables of a robot controller

Check Action
Is the communication permission correctly set? ("Read/Write" or "Read-only") Check the communication settings in TP.
Is any edit dialogue displayed in the teach pendant? Close the dialogue.
Is any error message displayed in the teach pendant? Clear the error.
Is the ORiN extension set? Add the ORiN option.
Is the variable name correctly set? Check the variable name.
In case of writing, is not the variable read-only? Check the variable and I/O attribute.
Is the range of index correctly specified? Check the range.

Trouble in Robot Moving

Check Action
Is the "Executable token" for a robot correctly set? (ANSI: Single point of control) Check the settings.
Is the "RobSlave" program running? Check the condition.
Does RobSlave.pac match the version of "NetwoRC Provider"? Use the modules included in the same ORiN Package.
Are the command names and parameters correctly specified? Check the command (function) specification

How to write data in an error state

Issue Possible Cause Solution
NetwoRC controller rejects all writing data request (ex. Variable, I/O) when an error occurred because of safety reason The controller restricts write operations in an error state for safety reasons.. 1.Enable a supervisory task (PAC) at controller startup.
2.Create a supervisory task program (TSR1, TSR2, etc.) that allows writing during an error state.
3.Invoke the supervisory task in the controller.
4.Send a notification from an external PC to the supervisory task to trigger the write operation.
5.Ensure the supervisory task is correctly defined and active before an error occurs.

Troubleshooting Hardware Issues:

Issue Possible Cause Solution
Unable to communicate with the Robot Controller Incorrect communication device selected (other than Ethernet). 1.Open System Manager in WinCAPS III.
2.Click on Set Communication and select the correct device.
Timeout error while communicating with the controller The timeout value is too low, or network latency is high. 1.Go to Communication Settings.
2.Increase the Timeout value (default is 4000 ms).
3.Check network stability and try again.
4.Increase the Retry count (default is 5).
5.Check Error Log
