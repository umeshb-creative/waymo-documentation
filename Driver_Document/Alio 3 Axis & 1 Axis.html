Alio ACS Motion Stage Controller Driver Development Guide

Introduction

Brief overview of the project

This project focuses on developing drivers for the Alio Stage utilizing the ACS motion controller in automation applications. The primary goal is to achieve seamless integration with a user interface (UI) for potential future use, facilitating precise motion control and positioning.

The ACS motion controller is responsible for powering the motors and managing movement across each axis. Key functionalities include trajectory planning, velocity profiling, and synchronization, ensuring smooth and accurate control of motion. The successful integration of these drivers will enhance the usability of the ACS motion controller in automated workflows, increasing efficiency and productivity across various industrial applications.

Purpose of the document

The purpose of this document is to provide a comprehensive guide for developed drivers for the Alio 3-axis stage (X, Y, R) and the Alio single-axis stage (R), using the ACS motion controller (specifically the SPiiPlusEC controller and UDMNT amplifier). The drivers will facilitate communication between the motor and external software applications, enabling precise control over various parameters such as position, velocity, acceleration, direction, Return home functionality etc..

The primary objective of the document is to provide a comprehensive guide for developed drivers Alio 3 axis stage (X,Y,R) and Alio single-axis stage (R) using ACS motion controller (SPiiPlusEC controller and UDMNT amplifier) used in automation using a UI application. This includes detailed instructions, software installation guide, a device description with its features, and best practices to ensure the successful development and integration of drivers with the devices.

Scope and objectives

  1. Driver Development: The document focuses on the development of a driver for Alio 3 axis stage (X, Y,R) and Alio Single Axis (R) using ACS motion controller using industry-standard protocols.
  2. Technical Information: It provides the technical specifications of Alio, hardware and software requirements, and detailed information about the Software Development Kit (SDK) provided for the device.
  3. SDK Overview: Introduction to SPiiPlus MMI Application Studio, covering the installation process and usage of these tools for development.
  4. Driver Implementation: This section covers the driver development process, including the selection of a suitable programming language, usage of APIs for interacting with the Alio ACS, limitations of the driver, necessary precautions before execution, and the actual implementation strategy.
  5. Testing and Debugging: The scope extends to testing and debugging methodologies, ensuring the driver performs reliably and meets the required performance standards in real-time automation scenarios.
  6. Documentation and Support: This document provides information about documentation standards for the development process and available support resources for developers working on integrating the driver with the Alio ACS Motion Stage Controller.

Overall, the aim is to facilitate the successful development and implementation of the Alio 3 axis stage (X, Y,R) and Alio 1 axis stage (R) using ACS motion controller, ensuring it meets the required standards and serves its intended purpose effectively.

The SPiiPlus MMI Application Suite installation CD includes:

Firmware -System-level software that runs on the controller's MPU (motion processing unit). Each product is provided by ACS with the most updated firmware released version (unless otherwise is required).

SPiiPlus Simulator -Windows-based controller simulator that runs on the PC without any connected hardware.

gRPC API’s Description for Alio ACS

  1. ACSOpen

    Description for ACSOpen API: Establishes a connection to the Alio ACS, enabling communication and control.

    RPC Command :rpc OpenDevice(.generic.CommandIPRequest) returns(.generic.GenericDeviceModel) {}

    Request:
    message CommandIPRequest {
    string ip_address = 1;
    string com = 2;
    int64 port_address = 3;
    int32 device_id = 4;
    string file_path = 5;
    int64 device_model =6;
    optional string device_alias=7;
    int32 axis =8;
    }
    Parameter Description
    device_alias The unique device_alias used to refer the device
    ip_address Specifies the IP address of the device.
    com Indicates the communication protocol or interface to be used
    port_address Denotes the specific port address for the device on the communication interface.
    device_id Unique identifier for the device, used to distinguish it from other devices in the system.
    file_path The path to the file, which may include configuration, program, or data files for the device.
    device_model Represents the model number or type of the device being communicated with.
    device_alias A user-defined alias for the device, providing a more descriptive or convenient reference.
    axis Specifies the axis number (e.g., X, Y, Z) on which the command or operation will be performed.
    enum DeviceType{
    NONE = 0;
    SENSOR_3D = 1;
    MOTION_STAGE = 2;
    CAMERA = 3;
    IO_MODULE = 4;
    SAFETY_CONTROLLER = 5;
    ANIMATIC_MOTOR=6;
    }
    enum DeviceModel{
    DUMMY_MODEL = 0;
    ED527 = 1;
    ED588 = 2 ;
    JR3000 = 3;
    OPTOE = 4;
    BASLER = 5;
    SPiiPlusEC=6;
    GT6600 = 7;
    MOTOR = 8;
    }
    enum DeviceStatus{
    OFFLINE = 0;
    ONLINE = 1;
    NOT_REACHABLE =2;
    }
    message CommandResponse{
    int32 resp_code = 1;
    string resp_message = 2;
    }
    Parameter Description
    resp_code Response code from the device API
    resp_message Response message code
    Response
    message CommandResponse {
    int32 resp_code = 1;
    string resp_message = 2;
    }
    Parameter Description
    resp_code Response code from the device API
    resp_message Response Message
    CLI Command: python alioacs.py ACSOpen 100.0.0.1 18 3 ALIO_ACS_1
    Description for CLI Command
    • OpenDevice: API called to the Alio ACS through CLI Command to connect and initialize all resources
    • 100.0.0.1: IP Address of the device
    • 18: Enum value representing the device type (18 for 3-Axis Alio, 6 for Single-Axis Alio).
    • 3:Specifies that it is a 3-Axis Alio (0 for Single-Axis Alio).

GLOSSARY

  • Controller: The electronic component that manages and controls the operation of the servo motor.
  • Encoder: A device used to measure the position or speed of a motor shaft and provide feedback to the controller.
  • Axis: In the context of motors and motion control, an axis refers to a direction or degree of freedom along which a motor or a mechanism can move.
  • Driver: A component that amplifies the control signals from the controller to the motor.
  • Motion Profile/PTP Movement: A predefined pattern of motion used to control the movement of the motor.
  • Acceleration: The rate of change of velocity of the motor.
  • Deceleration: The rate at which the motor slows down.
  • Torque: The rotational force produced by the motor.
  • Feedback: Information about the motor's performance that is fed back to the controller for adjustment.
  • Position Control: The ability to control the motor's position accurately.
  • Speed Control: The ability to control the motor's speed accurately.
  • Motion Control: The ability to control the motor's motion, including speed, position, and acceleration.
  • Integration: The process of incorporating the motor into a larger system or machine.
  • Communication Protocol: The rules and conventions for communication between the motor and other devices systems.
  • EtherCAT: A real-time Ethernet fieldbus system used for communication in industrial automation.
  • CANopen: A communication protocol based on the Controller Area Network (CAN) bus standard.
  • Modbus: A communication protocol commonly used in industrial automation systems.
  • PLC (Programmable Logic Controller): A digital computer used for automation of electromechanical processes.
  • HMI (Human Machine Interface): A device or software that allows users to interact with the motor or control system.
