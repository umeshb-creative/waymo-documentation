How to create a new flow in node-red?

One has to just click on the plus icon where a flow could be added as shown below in the screenshot.

Description text

After clicking on the plus icon, you will see Flow 1 with default name one tab will be created just double click on it and you will have the provision to rename the flow with name and description as shown below. Once you enter the name and description just click on the done button and then on deploy button to save the changes on node-red.

Description text

Summary:

  • Click on the + icon to create a new tab named Flow 1 by default.
  • Double-click on the newly created tab to open a popup where you can enter a custom name and description.
  • After entering the details, click on the Done button.
  • Click on the Deploy button to save the changes in Node-RED.
  • Once saved, drag and drop the required node blocks and connect them using connectors as per the requirement.

How to create one basic flow for setting an IO of a brainbox?

Look for the category IO Module you will see all the devices api node blocks which falls under IO Module category as shown in the below screenshot.

Description text

You can drag and drop any node block from left hand side to your sample flow window based on your flow / business requirement. Here am dragging the Open Device node block which will establish connection with brainbox. So, each node block you see under IO Module category all those are reusable flows. What is Reusable flow and how it can be created will discuss later in a separate section under this documentation.

Now we can double click on the node block and click on Edit subflow template as shown below.

Description text

After clicking on Edit Subflow template you will see the actual flow inside this subflow where the node js code is been written to call the C# gRPC api

Description text

You can just double click on Open Device function node block to view the code.

Description text

Here you can see this function node or the reusable node block (. i.e., Brainbox Open device) takes two arguments like IP Address and Device Alias. So, if you want to execute this reusable flow from an inject node you can use below approach / configuration.

In below screenshot we have shown how we can configure the inject node which act as an input node for the open device. So, in the left-hand side panel you will have to search for the inject node and then drag it on the flow and double click on it and give msg.IpAddress and msg.DeviceAlias as left-hand side operator and right-hand side as String where you will give the actual value.

Refer the below screenshot for the same:

Description text

After configuration click on Done button and then just connect the inject block with Brainbox Open device node block along with Debug node. Debug node is used to see the output in the debug window on the right-hand side. Lastly click on the start of the inject block so it will start executing the flow and output will be seen on the right-hand side. Refer below screenshot.

Description text

Now you can drag and drop the SET IO node block from IO Module which internally calls SET IO Api of brainbox device. SET IO node blocks needs input as IO State, IO Number and Device Alias as shown below

Description text

Summary:

  • Search for the inject node on left-hand side and drag it on the flow window.
  • Configure the msg.payload with input data in Json format.
  • Search for the Open device on left-hand side and drag it on the flow window.
  • Connect the connector from Inject node to Open Device node if required you can connect the debug node as well to view the output on the right -hand side window.
  • Search for the Change node and SETIO node block on left-hand side and drag it on the flow window.
  • Configure the msg.payload with input data in Json format inside the change node block.
  • Connect the connecter from Change node block to Open device
  • Execute the complete flow.

How to create one basic flow for running a program via Janome Robot?

Search for inject node block first to configure the input values to execute the node block then search for Open device for Janome under Motion Stage category drag and drop both node blocks in the flow and start configuring the inject node and later connect it with the open device node block as shown in the below screen shot.

Description text

After configuring this flow search for Run program node block under the motion stage category to run the program.

Here we need to add Change node block to set the input since in this flow we need to provide multiple inputs to multiple node blocks. If we have only single input throughout the flow we can use inject node block else we need to go with change node block and configure our input variables with values.

As shown below we have configured the Device alias and program number input for executing run program api via node-red flow.

How to create XML file for MFG Upload?

You will have to first drag and drop Initialize XML file reusable node block which accepts below inputs

  • msg.payload.unit_serial
  • msg.payload.station_type
  • msg.payload.operator

You will have to pass these inputs either from Dashboard Form control or via Inject node.

This Initialize XML node block will create an xml file in one folder. Now folder path is configurable as shown in the screenshot. The value is configurable under settings -> environmental refer Result_FilePath.

Description text

User can configure any path with file name under Result_FilePath or he can add new variable with value by just clicking on +add button in the left-hand side bottom of the popup.

Next to generate the Test parameters and other failure codes in the XML one has to drag Generate MFG XML File node block and in case of failure Generate MFG XML File FAIL Case as shown below.

Description text

Here we have hardcoded the input for the XML file in JSON array structure same can be passed via any script for example python script should return data in JSON array structure only since this reusable block supports input in JSON array structure. Here we have used json parser node block which will convert the json data to json string with indentation. Make sure you check mark the checkbox for Format JSON String.
