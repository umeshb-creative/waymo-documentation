Troubleshooting Guide for Basler Cameras: ace2 acA2500-14gm & a2A2448-23gmPRO

This guide helps diagnose and resolve common hardware, desktop application, and driver issues with the Basler Cameras(ace2 acA2500-14gm & a2A2448-23gmPRO)

Follow the steps below to check for power, connection, overheating, and network-related problems.

Hardware-Trouble Shooting Guide

Understanding LED Indicators and Troubleshooting Power Issues:

Issue Possible Cause Solution
LED Off 1.The power adapter not connected or is faulty.
2.Insufficient Power.
3.Power over Ethernet (PoE) not meeting IEEE 802.3af standard.		 1.Ensure the power adapter is properly plugged in and working.
2.Try a different adapter.
3.Check if the PoE switch or injector meets IEEE 802.3af specifications.
4.Use a compatible power source.
Flashing Bright Green (System Error) Internal hardware fault. Power cycle the camera. If the issue persists, contact Basler support.
Flashing Bright Green (3x long),Repeatedly until Error is corrected User error (e.g., temperature issue). Check environmental conditions and ensure the camera is operating within the recommended temperature range.
Alternating Green/Dark Green IP address allocation issue. Set the camera to a Static IP using Pylon IP Configurator.
Solid Green with periodic dark phase IP conflict on the network. 1.Assign a unique IP address to the camera.
2. Avoid duplicate IPs.
Alternating: Bright green / off Link down (powered, but no GigE connection) 1.Check the Ethernet Cable.
2.Verify the Network Port.
3.Check for Loose or Damaged Connectors.
4.Ensure the Network Interface Card (NIC) is Enabled.
5.Check for IP Conflicts.
6.Update Network Drivers & Firmware.

Note:

1.The Basler Camera LED will remain solid green when it is properly powered and has successfully established a network connection.

2.Electrical Requirements: Power Supply to Basler Camera via Power over Ethernet (PoE)- Power must comply with the IEEE 802.3af specification. More Info on Electrical Requirements can be found Here

3.Power Requirements for Cable and I/O Requirements for Basler Camera can be found here

Connection or Network Issues Troubleshooting:

Issue Possible Cause Solution
Camera not Recognized in Windows Device Manager Driver not installed. Install or update the Basler Pylon Driver from the official Basler website.
Network Connection Lost (Alternating Bright Green/Off LED) No GigE connection. Connect the camera directly to the PC to rule out switch issues. Check for firewall restrictions.
Camera Not Accessible via IP IP conflict or wrong subnet. Use ipconfig or ping to verify the assigned IP address. Set the Jumbo Frame size to a maximum (1.5kB) for better communication.
Camera Not Detected in Pylon Viewer 1.IP address allocation issue.
2.Loose or damaged Ethernet cable.		 1.Set the camera to a Static IP using Pylon IP Configurator.
2.Check and replace the Ethernet cable. Ensure the cable supports GigE Vision.

Physical Damage & Overheating Troubleshooting:

Issue Possible Cause Solution
Camera Overheating. Poor ventilation or high ambient temperature. Ensure proper cooling. Use the camera in a well-ventilated area.
Blurry Images / Focus Issues. Dust or debris on the lens. Clean the lens carefully with a microfiber cloth. Remove the lens cap before use.

Note:

Safety Instructions can be found Here

Network & Communication Issues TroubleShooting:

Issue Possible Cause Solution
IP Address Conflict Multiple devices using the same IP. Use Pylon IP Configurator to assign a unique IP or use ipconfig to eliminate any conflicts
Slow Image or Incomplete Image Transfer Incorrect frame size settings. Set Jumbo Frames (Jumbo Packets) to maximum size (1.5kB).
Firmware Compatibility Issue Outdated firmware/software. Upgrade firmware using Basler Pylon Firmware Upgrader.

Reset & Recovery:

Step Action
Power Cycle Turn off the camera, wait for 10 seconds, and turn it back on.
Factory Reset Use the Pylon Viewer Software Suite to reset the camera to factory settings.

Trouble Shooting Desktop Application-Related Issues:

Issue Category Troubleshooting Steps
Driver Installation Failed 1.Verify that the Pylon Viewer SDK and drivers are installed and up to date
2.Latest Pylon Viewer Software Suite Can be Found Here
3.Check for driver conflicts in Device Manager (Windows)
4.Reinstall drivers and restart the system.
Compatibility Issues 1.Ensure the software version supports the camera model
2.Check OS compatibility (Windows). Check for System Requirements and Installation Here
3.Verify required dependencies/libraries for the software
Configuration Issues 1.Confirm that camera configurations (e.g., Baud Rate, IP Address) match requirements
2.Basler Camera Network Configuration Settings can be found Here
3.Check if the correct communication protocol is used (GigE)
4.IP Configuration should be Static
5.Use Pylon IP Configurator to address any IP issues for the camera
6.Verify firmware compatibility with the software version
7.Firmware Can be Upgraded using Firmware Upgrader in Pylon Viewer Software Suite

Troubleshooting Driver-Related Issues:

Below Table shows the Challenges, Function Calls, and Troubleshooting Steps for Basler Camera Driver Development.

Issue API/Function name Solution Note
Basler Camera Not Connected OpenDevice 1. Verify Basler Camera connection using Pylon Viewer Software Suite
2.Ensure the connection is closed before calling OpenDevice API through gRPC
3.Check for OpenDevice API timeout errors or specific error codes
4.Refer to Error Codes and Descriptions for further troubleshooting
5.Close the Connection before calling OpenDevice API through gRPC
Lost Packets/Frames in GigE Cameras GetImage 1.Switch to Guru level in Pylon Viewer
2.Optimize Network Adapter Settings:
- Enable Jumbo Frames (maximum value).
- Increase Receive Descriptors to the maximum value.
- Set Interrupt Moderation Rate to off or lowest value.
3. Increase Inter-Packet Delay for stable frame rate.
Error Code 3774873620 (0xE1000014) - Incomplete Grab GetImage 1.Indicates lost/corrupted packets due to hardware issues (NIC, switch, Ethernet cable)
2.Increase Inter-Packet Delay to reduce bandwidth usage
3.Enable Jumbo Frames on NIC or switch (if supported)
4. If Jumbo Frames are not supported, set Packet Size = 1500
Error: Node Sequence Enable Is Not Writable Set and Get Gain, Set and Get Exposure Time, Set and Get Pixel Format 1.The Sequence Enable parameter to enable/disable the Sequencer is only writable when all auto functions (Gain Auto, Exposure Auto) are set to Off
2.The node is also not writable while grabbing images
3.If the camera doesn't support the Sequencer feature at all, the node is also not writable
4.Note that the node name and the variable type are different for GigE
Running Different Pylon Versions 1.Installing a new Pylon version uninstalls the old version
2.Use the Copy Deployment feature (Pylon 5 or later) to run multiple versions side by side
Some Nodes are not supported in New versions of Pylon Viewer. For Example, Pylon Viewer 7.5.0.15 doesn’t support ExposureTimeAbs.GetValue(), while 3.3.0.15479 does.
Windows Compatibility for Pylon 1.Pylon 5+ supports Windows 7 and higher
2.Pylon 6.2+ does not support Windows 7
3.Windows 10 is supported only by Pylon 5 and higher
Overtrigger Error (Changing Exposure During Grabbing) Ensure exposure time changes follow the recommended guidelines
Black Level Adjustment BaslerLightControl Adjust the Black Level for better contrast and visibility in dark areas.
Pylon Viewer will not Open Camera Connection OpenDevice and CloseDevice 1.If the camera is not closed properly—either due to the Close API not being called or an Unexpected Service crash—the camera handlers may not be released correctly. As a result, the camera will not allow a new connection until the existing handlers are properly released.
2.Check the camera connection in Pylon Viewer
3.Restart the PC/Station for resolving this Issue OR Wait until pylon viewer Connects to the camera
4.Restart the application or release the camera from previous sessions
5.Wait for Pylon Viewer to automatically regain connection. In cases of improper handling, this process can take 10 to 15 minutes.
Missing dependent dlls / Driver's C++ library dll won't load OpenDevice and CloseDevice Install MSVC v143 –VS 2022 C++ x64/x86 build tools (v14.36 -17.6)
C++ v14.36 (17.6) MFC for v143 build tools (x86 & x64)
C++ v14.36 (17.6) ATL for v143 build tools (x86 & x64) packages in PC that support Basler Camera.
