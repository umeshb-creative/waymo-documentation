Triton Camera driver document

Introduction

Brief overview of the Project

This project focuses on developing drivers for the Triton Camera (TRI0245S-MC) for use in automation applications, with an emphasis on integrating it with a user interface (UI) application. The goal is to create software components that facilitate smooth communication between the Triton Camera, enabling precise control over camera settings, image capture, and real-time data streaming. By integrating these systems, the project aims to improve the operational efficiency and functionality of automated processes, contributing to increased productivity and reliability in industrial environments.

Purpose of the document

The purpose of this document is to provide a detailed guide for developing a driver for the Lucid Triton Camera (TRI0245S-MC). The driver will enable communication between the Camera and external software application (Arena View), allowing to capture an Image, change camera settings like pixel format, exposure time, gain and many more. It covers the software development process, including setting up the development environment, implementing the driver, testing and debugging, and integrating the driver with the Lucid Triton Camera.

The primary objective of the document is to provide a comprehensive guide for developed drivers for Lucid Triton Camera (TRI0245S-MC) used in automation using a UI application. This includes detailed instructions, software installation guide, a device description with its features, and best practices to ensure the successful development and integration of drivers with the devices.

Scope and objectives

  1. Driver Development: The document focuses on the development of a driver for Triton Camera using industry-standard protocols.
  2. Technical Information: It provides the technical specifications of the camera, hardware and software requirements, and detailed information about the Software Development Kit (SDK) provided for the device.
  3. SDK Overview: Introduction to Arena SDK and Arena View software, covering the installation process and usage of these tools for development
  4. Driver Implementation: This section covers the driver development process, including the selection of a suitable programming language, usage of APIs for interacting with the camera, limitations of the driver, necessary precautions before execution, and the actual implementation strategy.
  5. Testing and Debugging: The scope extends to testing and debugging methodologies, ensuring the driver performs reliably and meets the required performance standards in real-time automation scenarios.
  6. Documentation and Support: : This document provides information about documentation standards for the development process and available support resources for developers working on integrating the driver with the Lucid Triton Camera.

Overall, the aim is to facilitate the successful development and implementation of the Triton Lucid Camera, ensuring it meets the required standards and serves its intended purpose effectively.

Device Overview

Description of the device for which the driver is being developed

LUCID Vision Labs Triton™ GigE Power over Ethernet (PoE) Cameras provide exceptional price performance in a compact 29 x 29mm form factor. These cameras are available with Sony’s Pregius™ and Starvis™ imaging sensors with up to 24.5 megapixels of resolution, as well as Sony’s polarized sensor. Each imaging sensor is actively aligned during manufacturing, ensuring sharp and crisp images even in the corners. Using the accessory IP67 lens tubes (sold separately), LUCID Vision Labs Triton GigE Power over Ethernet (PoE) Cameras can operate in harsh environments where dust or water are present up to IP67 thresholds. Triton cameras are ideal for automotive, process control, and food and beverage applications..

Software Overview and Requirements – Arena SDK

The Arena SDK is a comprehensive software development kit designed for high-performance imaging applications. It provides developers with a powerful set of tools for working with cameras, such as those in the Lucid Vision product line, including the Triton Camera series. The SDK simplifies integration, control, and management of camera operations for various industrial and automation applications.

Features of Arena SDK

  • Cross-Platform Compatibility:Arena SDK supports both Windows and Linux operating systems, allowing developers to build applications across different platforms.
  • Easy Camera Integration: It provides simplified APIs for connecting and managing cameras, making it easy to control settings such as exposure time, gain, frame rate, and pixel format.
  • GenICam Compliance: Arena SDK is fully compliant with the GenICam standard, allowing seamless control of various cameras that adhere to this standard.
  • High-Speed Data Streaming: Designed for high-performance applications, the SDK supports efficient data transfer and streaming from cameras to external applications.
  • Extensive Documentation and Sample Code: Arena SDK offers extensive documentation, tutorials, and sample applications to help developers quickly understand and implement key functionalities.
  • Multi-Camera Support: The SDK allows simultaneous operation of multiple cameras, making it ideal for complex imaging systems.
  • Advanced Imaging Controls: Provides features like region of interest (ROI), binning, and trigger modes, which are crucial for optimizing performance in specific applications.

Hardware and Software Requirements

Software Requirements:

  • Operating Systems: Windows 10/11, Linux distributions such as Ubuntu.
  • Supported Programming Languages: The SDK supports C, C++, and C# languages, providing flexible options for developers.
  • Libraries: Includes precompiled libraries and drivers required for camera operation and integration with custom software applications.
  • Development Environment: Compatible with popular IDEs such as Microsoft Visual Studio and GCC-based compilers on Linux.
  • Power supply for the camera- Power Over Ethernet

Hardware Requirements:

  • Lucid Vision Cameras: Designed for use with Lucid Triton Camera and other models in the Lucid Vision series
  • High-Performance Processors: For real-time image processing and handling high data throughput, a multi-core CPU is recommended
  • Memory and Storage: Sufficient RAM and storage to handle large image data files and high-speed data streaming.

Additional Requirements:

This version of Microsoft Visual Studio works best with Internet Explorer 10 or higher. If Internet Explorer 10 is not installed on your computer, some features might not work as expected.

gRPC API’s Description for lucid triton Camera

  1. OpenDevice

    Description for OpenDevice API: Establishes a connection to the Triton Camera, initializing communication for further operations.

    RPC Command: rpc OpenDevice(.generic.CommandIPRequest) returns(.generic.GenericDeviceModel) {}

    Request:
    message CommandIPRequest {
    string ip_address = 1;
    string com = 2;
    int64 port_address = 3;
    int32 device_id = 4;
    string file_path = 5;
    int64 device_model =6;
    optional string device_alias=7;
    int32 axis =8;
    }
    Parameter Description
    device_alias The unique device_alias used to refer the device
    ip_address Specifies the IP address of the device.
    com Indicates the communication protocol or interface to be used
    port_address Denotes the specific port address for the device on the communication interface.
    device_id Unique identifier for the device, used to distinguish it from other devices in the system.
    file_path The path to the file, which may include configuration, program, or data files for the device.
    device_model Represents the model number or type of the device being communicated with.
    device_alias A user-defined alias for the device, providing a more descriptive or convenient reference.
    axis Specifies the axis number (e.g., X, Y, Z) on which the command or operation will be performed.
    enum DeviceType{
    NONE = 0;
    SENSOR_3D = 1;
    MOTION_STAGE = 2;
    CAMERA = 3;
    IO_MODULE = 4;
    SAFETY_CONTROLLER = 5;
    ANIMATIC_MOTOR=6;
    }
    enum DeviceModel{
    DUMMY_MODEL = 0;
    ED527 = 1;
    ED588 = 2 ;
    JR3000 = 3;
    OPTOE = 4;
    BASLER = 5;
    SPiiPlusEC=6;
    GT6600 = 7;
    MOTOR = 8;
    }
    enum DeviceStatus{
    OFFLINE = 0;
    ONLINE = 1;
    NOT_REACHABLE =2;
    }
    message CommandResponse{
    int32 resp_code = 1;
    string resp_message = 2;
    }
    Parameter Description
    resp_code Response code from the device API
    resp_message Response message code
    CLI Command: python tritoncamera.py OpenDevice 192.168.200.100 Triton
    Description for CLI Command
    • OpenDevice: API called to the Triton Camera through CLI Command to Connect and Initialize all Resources
    • 192.168.200.100: IP Address of the device
    • Triton:Aliased Name for the Device

    GLOSSARY

    Glossary of Terms for Triton Camera:

    • Driver: A software component that enables communication between the operating system and a hardware device, in this case, a camera.
    • SDK (Software Development Kit): A set of tools, libraries, and documentation that developers use to create software applications for a specific platform or framework.
    • API (Application Programming Interface): A set of rules and protocols that allow different software applications to communicate with each other.
    • Firmware: Software that is embedded into a hardware device, such as a camera, to control its operation.
    • Resolution: The number of pixels in an image, typically measured in megapixels (MP), that determines the level of detail and clarity in the image.
    • Frame Rate: The number of frames or images displayed or processed per second, often measured in frames per second (fps).
    • Gain: Gain in the context of a camera refers to the amplification of the signal from the camera sensor. However, increasing the gain too much can introduce noise into the image, reducing its quality. Gain is usually measured in decibels (dB).
    • Exposure Time: : The amount of light that reaches the camera sensor, controlled by settings such as shutter speed, and aperture.
    • Pixel: The smallest unit of a digital image, representing a single point of color.
    • Buffer: A temporary storage area in memory used to hold data, such as images, before it is processed or transferred.
    • Streaming: The process of continuously transmitting data, such as video, over a network in real-time.
