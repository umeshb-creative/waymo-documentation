    LMI Gocator driver document

    Introduction

    Brief overview of the project

    This project aims to develop drivers for the Gocator 2530 and 3506 high-performance 3D smart sensor, integrating it with a user interface (UI) application for use in automation. The goal is to enable seamless communication between the sensor and the UI application for inline inspection tasks. This integration will allow for capturing high-resolution 3D point cloud data of the target object, facilitating detailed surface analysis and measurement. Ultimately, the project seeks to enhance the usability and functionality of the Gocator 2530 and 3506 in automated processes, leading to increased efficiency and productivity in various industrial settings.

    Purpose of the document

    The purpose of this document is to provide a detailed guide for developing a driver for the Gocator 2530 and 3506 device . The driver will enable communication between the 3d Smart Laser Line Profile Sensor and external software applications, allowing to get raw 3D data points, Load and Save programs and many more. It covers the software development process, including setting up the development environment, implementing the driver, testing and debugging, and integrating the driver with the Gocator 2530 and and 3506.

    The primary objective of the document is to provide a comprehensive guide for developed drivers Gocator 2530 and and 3506 used in automation using a UI application. This includes detailed instructions, software installation guide, a device description with its features, and best practices to ensure the successful development and integration of drivers with the devices.

    Scope and objectives

    1. Driver Development: The document focuses on the development of a driver for Gocator 2530 and and 3506 using industry-standard protocols.
    2. Technical Information: It includes technical specifications of the device, hardware and software requirements, and information about the Software Development Kit (SDK) provided for the device.
    3. SDK Overview: Introduction to GoSDK and its Installation.
    4. Driver Implementation: It covers the development process, including the choice of programming language, information on APIs, how to use APIs, limitations of the driver, precautions to take before execution, and the actual implementation of the driver.
    5. Testing and Debugging: The scope extends to testing and debugging procedures to ensure the reliability and functionality of the driver for real-time application
    6. Documentation and Support: Lastly, the document includes information on documentation standards and support resources available for developers.

    Overall, the aim is to facilitate the successful development and implementation of the Gocator 2530 and 3506, ensuring it meets the required standards and serves its intended purpose effectively.

    Device Overview

    Description of the device for which the driver is being developed

    Gocator 2530 and and 3506 is a high-performance 3D smart sensor designed for industrial automation and quality control applications. It features a built-in web interface for easy setup and configuration, making it ideal for inline inspection tasks. The sensor uses laser triangulation technology to capture high-resolution 3D point cloud data of the target object, allowing for detailed surface analysis and measurement. With its compact design and versatile capabilities, the Gocator 2530 and 3506 is well-suited for a wide range of industries, including automotive, electronics

    Features of Gocator 2530 and 3506

    • Up To 100 Mm Field Of View
    • 28 Microns X Resolution
    • 0.5 Microns Z Repeatability
    • 10,000 Profiles Per Second Including 3d Measurement
    • Setup & Control Via Web Browser Or Sdk
    • Built-In Tools, No Programming
    • Extend With Gdk And Gomax

    Software Overview and Requirements - GoSDK

    The Gocator Software Development Kit (GoSDK) includes open-source software libraries and documentation that can be used to programmatically access and control Gocator sensors.

    Features Of GoSDK

    • Sensor Control: APIs for configuring sensor settings such as exposure, resolution, and measurement parameters.
    • Data Acquisition: Functions for capturing 3D point cloud data from the sensor.
    • Data Processing: Tools for processing and analyzing 3D point cloud data, including filtering, alignment, and measurement functions.
    • Integration: Support for integrating Gocator sensors into existing applications and systems, including PLCs and robotic systems.
    • Visualization: Functions for visualizing 3D point cloud data in real-time or offline for analysis and inspection.
    • Calibration: Tools for calibrating sensors and correcting for distortion or misalignment in the captured data.
    • Networking: APIs for communicating with Gocator sensors over Ethernet or other network protocols.

      Description for GetDeviceDetails API: Retrieves detailed information about the connected Gocator, including its configuration and status.

      RPC Command : rpc GetDeviceDetails(.generic. CommandRequest) returns (.generic.GenericDeviceModel) {}

      Request:
      message CommandRequest {
      string device_alias = 1;
      }
      Parameter Description
      device_alias Unique alias of the device
      Response:
      message GenericDeviceModel{
      string instance_id = 1;
      string device_alias = 2;
      DeviceType device_type = 3;
      DeviceModel device_model_name = 4;
      optional DeviceStatus device_status = 5;
      string ipaddr = 6;
      string mac_addr = 7;
      google.protobuf.Value conn_handle=8;
      string port_name = 9;
      int64 port_address = 10;
      CommandResponse cmd_response = 11;
      }
      Parameter Description
      instance_id New instance for the device in the enumeration
      device_alias device_alias Unique identifier (alphanumeric) for every device in the enumeration
      device_type device_type Indicates the type of device like Motion stage, Safety Controller etc (defined below in the DeviceType enum)
      device_model_name Indicate the model name for each device (define below in the DeviceModel enum)
      device_status device_status indicate whether the device is online , offline or not reachable (define below in the device status enum)
      ipaddr indicate the ipaddress of the device
      mac_addr mac_addr indicate the mac address of the device
      Conn_handle Conn_handle Handle object to keep the device open
      port_name Port name
      port_address port_address Port address
      cmd_response cmd_response Indicates command response that describes API response code and its description (Defined below in CommandResponse)

      NONE = 0;
      SENSOR_3D = 1;
      MOTION_STAGE = 2;
      CAMERA = 3;
      IO_MODULE = 4;
      SAFETY_CONTROLLER = 5;
      ANIMATIC_MOTOR=6;
      }
      enum DeviceModel{
      DUMMY_MODEL = 0;

      ED588 = 2 ;

    GLOSSARY

    • Gocator: A family of 3D smart sensors manufactured by LMI Technologies, used for inline inspection, measurement, and quality control in industrial applications.
    • 3D Point Cloud: A set of data points in a three-dimensional coordinate system that represents the external surface of an object scanned by a 3D sensor, such as the Gocator.
    • Field of View (FOV): The area that a sensor can capture or observe at any given moment. It is typically defined by the sensor's lens and sensor size.
    • Resolution: The smallest detectable change in position or dimension that a sensor can measure. Higher resolution sensors can detect smaller features.
    • Scan Rate: The speed at which a sensor captures and processes data. It is usually measured in Hz or frames per second (FPS).
    • Measurement Range: The range of distances over which a sensor can accurately measure objects. It is typically defined by the sensor's optics and laser.
    • Profile: A 2D cross-section of a 3D object captured by the sensor. Gocator sensors often capture profiles along a line to build a 3D point cloud.
    • ROI (Region of Interest): A specific area within the sensor's field of view where measurements or inspections are performed.
    • Calibration: The process of adjusting a sensor to ensure that its measurements are accurate and consistent. Calibration is often necessary to correct for distortions or misalignments.
    • Alignment: The process of adjusting the sensor's position or orientation to ensure that it is correctly aligned with the object being scanned.
