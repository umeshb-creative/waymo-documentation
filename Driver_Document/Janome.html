Janome driver document

Introduction

Brief overview of the project

This project focuses on developing drivers for Janome robots to be utilized in automation applications, with a particular emphasis on integrating them with a user interface (UI) application for future use. The project aims to create software components that enable seamless communication between Janome robots and the UI application, allowing for intuitive control and monitoring of the robotic systems. Through this integration, the project seeks to enhance the usability and functionality of Janome robots within automated processes, contributing to increased efficiency and productivity in various industrial settings.

Purpose of the document

The purpose of this document is to provide a detailed guide for developing a driver for the Janome device. The driver will enable communication between the robot and external software applications. It covers the software development process, including setting up the development environment, implementing the driver, testing and debugging, and integrating the driver with the Janome device.

The primary objective of the document is to provide a comprehensive guide for developed drivers for Janome robot used in automation using a UI application. This includes detailed instructions, code examples, and best practices to ensure the successful development and integration of drivers with the devices.

Scope and objectives

  1. Driver Development: The document focuses on the development of a driver for Janome device.
  2. Technical Information: It includes technical specifications of the device, hardware and software requirements, and information about the Software Development Kit (SDK) provided for the device.
  3. SDK Overview: There was no SDK available for Janome robot, so we have developed the entire driver code from scratch.
  4. Driver Implementation: It covers the development process, including the choice of programming language, information on APIs, how to use APIs, limitations of the driver, precautions to take before execution, and the actual implementation of the driver.
  5. Testing and Debugging: The scope extends to testing and debugging procedures to ensure the reliability and functionality of the driver.
  6. Documentation and Support: Lastly, the document includes information on documentation standards and support resources available for developers.

Overall, the aim is to facilitate the successful development and implementation of the Janome driver, ensuring it meets the required standards and serves its intended purpose effectively.

Device Overview

Description of the device for which the driver is being developed

The Janome Desktop Robot JR3000 Series represents a new generation of robots characterized by their affordability, efficiency, and functionality. These robots have been engineered to offer high performance at a low cost. One of the key achievements of this project is the successful reduction of price without compromising on essential functionalities.

A notable feature of these robots is the utilization of stepping motors in conjunction with specialized micro step driving circuits. This innovative combination serves to conserve energy and minimize the required installation space. By leveraging advanced technologies and optimization strategies, the JR3000 Series delivers optimal performance while remaining cost-effective and versatile.

Some of the applications of the Janome robot includes:

  • Liquid dispensing
  • Screw tightening and loosening
  • Automatic Tool Bit Replacement
  • Depaneling
  • Pick and place

Technical Specifications of Janome

LAN Communication Settings (IP Address Settings)

The settable items consist of the following three types:

  • IP Address
  • Subnet Mask
  • Default Gateway

The following values are set to the robot by default:

  • IP Address: 192.168.200.180
  • Subnet Mask: 255.255.255. 0
  • Default Gateway: 0. 0. 0. 0

COM Communication Settings

The settable items consist of the following

  • Baud rate: 2400/4800/9600/19200/38400/57600/76800/115200
  • Character length: 8 bit/7 bit
  • Stop bit: 1 bit/2 bit
  • Parity: None/Even parity/ Odd parity

The default values for the communication settings are as follows:

  • Baud rate 9600
  • Character length 8 bit
  • Stop bit 1 bit
  • Parity None

DRIVER DEVELOPMENT

Introduction to driver development for the device

  • JanomeDriverAPIs.py : JanomeDriverAPIs.py serves as the core interface, offering APIs that encapsulate the minimum functionalities required by Janome devices, as specified by Waymo. These APIs enable developers to interact with Janome devices programmatically, accessing essential functionalities with ease.
  • JanomeProtocol.py : JanomeProtocol.py plays a crucial role in establishing communication between the software and Janome devices. It provides APIs to connect to the device, send and receive packets, and encode/decode data according to the Janome protocol. This component ensures reliable and efficient data transmission, facilitating seamless interaction between the software and Janome devices.
  • JanomeData.py : JanomeData.py serves as a repository for storing all device-related details, including IP addresses, ports, and data commands. This centralized storage ensures that essential device information is readily accessible to the software, streamlining the development process and enhancing overall system efficiency.

Choice of programming language (Python or C++)

For developing the driver we have used Python.

gRPC API’s Description for Janome

  1. OpenDevice

    Description for OpenDevice API: To establish the connection with Janome.

    RPC Command :
    rpc OpenDevice(.generic.CommandIPRequest) returns(.generic.GenericDeviceModel) {}
    Request:
    message CommandIPRequest{
    string ip_address = 1;
    string com=2;
    int64 port_address=3;
    }
    Parameter Description
    ip_address ip_address IP Address of the Device
    com COM Port Address of the Device with serial communication(ex. COM1) Not applicable for Janome
    port_address Port address of the device with serial communication (ex. 1)
    Response
    message GenericDeviceModel{
    string instance_id = 1;
    string device_alias = 2;
    DeviceType device_type = 3;
    DeviceModel device_model_name = 4;
    optional DeviceStatus device_status = 5;
    string ipaddr = 6;
    string mac_addr = 7;
    google.protobuf.Value conn_handle=8;
    string port_name = 9;
    int64 port_address = 10;
    CommandResponse cmd_response = 11;
    }
    Parameter Description
    instance_id New instance for the device in the enumeration
    device_alias Unique identifier (alphanumeric) for every device in the enumeration
    device_type Indicates the type of device like Motion stage, Safety Controller etc (defined below in the DeviceType enum)
    device_model_name Indicate the model name for each device (define below in the DeviceModel enum)
    device_status Indicate whether the device is online , offline or not reachable (define below in the device status enum)
    ipaddr Indicate the ipaddress of the device
    mac_addr Indicate the mac address of the device
    Conn_handle Handle object to keep the device open
    port_name Port name
    port_address Port address
    cmd_response Indicates command response that describes API response code and its description (Defined below in CommandResponse)

    enum DeviceType{
    NONE = 0;
    SENSOR_3D = 1;
    MOTION_STAGE = 2;
    CAMERA = 3;
    IO_MODULE = 4;
    SAFETY_CONTROLLER = 5;
    ANIMATIC_MOTOR=6;
    }
    enum DeviceModel{
    DUMMY_MODEL = 0;
    ED527 = 1;
    ED588 = 2 ;
    JR3000 = 3;
    OPTOE = 4;
    BASLER = 5;
    SPiiPlusEC=6;
    GT6600 = 7;
    MOTOR = 8;
    }
    enum DeviceStatus{
    OFFLINE = 0;
    ONLINE = 1;
    NOT_REACHABLE =2;
    }
    message CommandResponse{
    int32 resp_code = 1;
    string resp_message = 2;
    }
    ParameterDescription
    resp_code resp_code Response code from the device API
    resp_message resp_message Description of the response code
    CLI Command: python janome.py OpenDevice 192.168.200.100
    Description for CLI Command
    • OpenDevice: OpenDevice is the name of the API.
    • 192.168.0.4: IP Address of the device

GLOSSARY

When developing a Janome driver, it's helpful to have a glossary of terms to ensure clarity and consistency in communication. Here's a glossary of terms commonly used in Janome driver development:

  • Janome Driver: Software module or library responsible for interfacing with Janome robotic systems, facilitating communication and control.
  • Robot Controller: Device or software component that manages the operation of a Janome robot, including motion control, program execution, and I/O handling.
  • API (Application Programming Interface): Set of protocols, tools, and definitions that allow software applications to communicate and interact with the Janome driver.
  • Command: Instruction or directive sent to a Janome robot through the driver to perform specific actions such as movement, I/O control, or program execution.
  • Response: Feedback or acknowledgment received from the Janome robot after executing a command, indicating the success or failure of the operation and providing relevant information or data.
  • Motion Control: Functionality within the Janome driver responsible for controlling the movement of the robot, including linear motion, joint motion, and velocity control.
  • I/O (Input/Output): Interface for interacting with external devices or sensors connected to the Janome robot, allowing the driver to read inputs and control outputs.
  • Program Execution: Process of running predefined motion sequences or routines on the Janome robot, typically stored in the robot controller's memory or loaded from external sources.
  • Configuration: Settings and parameters that can be adjusted or customized within the Janome driver to tailor its behavior to specific requirements or environments.
  • Error Handling: Mechanism for detecting, reporting, and managing errors or exceptions that occur during interaction with the Janome robot, ensuring robustness and reliability.
  • Coordinate System: Reference frame used to specify the position and orientation of the Janome robot's end-effector or individual joints, facilitating precise motion control and trajectory planning.
  • Velocity: Speed at which the Janome robot's end-effector or joints move during motion, typically measured in units per second (e.g., millimeters per second or degrees per second).
  • Acceleration: Rate of change of velocity over time, determining how quickly the Janome robot can accelerate or decelerate during motion, influencing performance and efficiency.
  • Positioning: Process of accurately locating and orienting the Janome robot's end-effector or individual joints at specific coordinates within its workspace, essential for precise manipulation tasks.
  • Feedback: Information provided by sensors or encoders on the Janome robot, used to monitor its state, verify motion accuracy, and adjust control parameters in real-time.
  • Interpolation: Algorithm used by the Janome driver to generate smooth and continuous motion trajectories between discrete points, ensuring seamless movement and avoiding jerky motion.
