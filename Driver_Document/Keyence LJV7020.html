Keyence LJV7020 driver document

Introduction

Brief overview of the project

This project focuses on developing drivers for LJV 7020 to be utilized in automation applications, with a particular emphasis on integrating them with a user interface (UI) application for future use. The project aims to create software components that enable seamless communication between LJV 7020 and the UI application, allowing fast, accurate, and reliable measurements for various surfaces and materials, making it suitable for applications such as quality control, dimensional measurement, and object detection and In-line measurements. Through this integration, the project seeks to enhance the usability and functionality of LJV 7020 within automated processes, contributing to increased efficiency and productivity in various industrial settings.

Purpose of the document

The purpose of this document is to provide a detailed guide for developing a driver for the LJV 7020 device. The driver will enable communication between the 3D Sensor and external software applications, allowing to Get Raw data, load and save program, start and stop storage and many more. It covers the software development process, including setting up the development environment, implementing the driver, testing and debugging, and integrating the driver with the LJV 7020.

The primary objective of the document is to provide a comprehensive guide for developed drivers LJV 7020 used in automation using a UI application. This includes detailed instructions, software installation guide, a device description with its features, and best practices to ensure the successful development and integration of drivers with the devices.

Scope and objectives

  1. Driver Development: The document focuses on the development of a driver for LJV 7020 using industry-standard protocols.
  2. Technical Information: It includes technical specifications of the device, hardware and software requirements, and information about the Software Development Kit (SDK) provided for the device.
  3. SDK Overview: Introduction to SMI Interface and its Installation.
  4. Driver Implementation: It covers the development process, including the choice of programming language, information on APIs, how to use APIs, limitations of the driver, precautions to take before execution, and the actual implementation of the driver.
  5. Testing and Debugging: The scope extends to testing and debugging procedures to ensure the reliability and functionality of the driver for real-time application.
  6. Documentation and Support: Lastly, the document includes information on documentation standards and support resources available for developers.

Overall, the aim is to facilitate the successful development and implementation of the LJV7020, ensuring it meets the required standards and serves its intended purpose effectively.

Device Overview

Laser profilers are laser displacement sensors that collect height data across a laser line rather than a single point. This enables 2D/3D measurements such as height difference, width, or angle to be performed using a single sensor. In addition to height data, KEYENCE profile sensors also collect intensity data to provide a stable solution for inline measurement and inspection. The line-up includes a wide range of sensors to support a variety of applications and industries. The World’s Fastest At 64,000 Profiles/Sec. Speed that makes the measurement of any product and of any shape a reality!

Description of the device for which the driver is being developed

LJV 7020 sensor is used for 3D measurement and inspection in line.

Following are the features of LJV 7020 :

  • Ultra-high-speed sampling the LJ-V Series is the fastest 2D laser measuring instrument in the world. This makes it possible to measure parts moving at extremely high speeds, in high definition, without missing a single one.
  • Profiles are accurately measured even in cases where black surfaces, inclines with low reflectivity and metallic surfaces with high reflectivity are mixed together under the same optical axis. Newly developed HSE3 -CMOS wide dynamic range has provided the LJ-V Series with improvements in both speed and detection stability Advanced gearing allowing pre-set traverse and take-up winding parameters including dwells and wrap counts.
  • The LJ-V Series is able to perform any measurement thanks to a variety of head variations and measurement modes, as well as 3D inspection when connected to an image processing system. In addition, the automatic setting optimization function makes operation easy for any user. Stand-alone, multi-axis linear interpolation with as many as 120 Smart Motors at a time.
  • Virtual-axis control into camming through gearing, which enables stand-alone, multi-axis coordinated motion Programmable and non-programmable protection features (both hardware and software).
  • Corrects positional misalignment of the target, eliminating errors in the measurement results due to target tilt. Accurate measurements are possible even in cases where the target is moving at random or when it is difficult to perform positioning.
  • The LJ-V7000 Series is equipped with a new function that makes it possible to individually set corrections on two separate features. This is effective when measuring gaps, angles, or height differences of two targets that move independently.
  • By understanding the positional relationship of both heads, it is possible to match the θ adjustment for both heads. Even when measuring targets with variation or incline changes, it is possible to measure the correct points.

DRIVER DEVELOPMENT

Introduction to driver development for the device

The LJ-V7000 Series communication library provides a communication interface for controlling the LJ-V7000 Series from a user application (Win32 DLL). For specific ways to use the communication library, refer to the sample program.

Execution environment

This section describes the necessary environment to execute applications that use the LJ-V7000 Series communication library.

  • Microsoft C runtime library The Microsoft C runtime library is required for the DLL to operate. Run vcredist_x86.exe included on the installation media to install the library.
  • Microsoft .NET Framework The Microsoft .NET Framework is required to run the sample application. Run NetFx20SP2_x86.exe included on the installation media to install the library.

Incorporating the Libraries

The files required at execution are listed below. Place these folders/files in the same folder as the executable file.

  • LJV7_IF.dll

Linking

  • C++: Linking The library can be linked implicitly or explicitly. To implicitly link the library, link with "LJV7_IF.lib". * "LJV7_IF.lib" was built with Visual C++ 2008 SP1.
  • Include files Include the following header files in the necessary source files.
  • LJV7_IF.h
  • LJV7_ErrorCode.h
  • C#/VB.NET Call each interface using the DllImport attribute. When passing a structure as an interface argument, specify the StructLayout attribute and pass a structure of the same memory structure as the DLL.

Steps To Be Carried Out

To build powerful Windows ljv 7020 applications carry out the following steps:

  1. Install the Smart Motor Interface Software Suite SDK.
  2. Install the desired free Microsoft Visual Studio tool.
  3. Prepare the C++ projects.
  4. Run the project.

gRPC API’s Description for LJV 7020

  1. OpenDevice

    Description for OpenDevice API: To Connect to the LJV Device.

    RPC Command :
    rpc OpenDevice(.generic.CommandIPRequest) returns(.generic.GenericDeviceModel) {}
    Request:
    message CommandIPRequest{
    string ip_address = 1;
    string com=2;
    int64 port_address=3;
    }
    Parameter Description
    ip_address ip_address IP Address of the Device
    com COM Port Address of the Device with serial communication(ex. COM1)
    port_address Port address of the device with serial communication (ex. 1)
    Response
    message GenericDeviceModel{
    string instance_id = 1;
    string device_alias = 2;
    DeviceType device_type = 3;
    DeviceModel device_model_name = 4;
    optional DeviceStatus device_status = 5;
    string ipaddr = 6;
    string mac_addr = 7;
    google.protobuf.Value conn_handle=8;
    string port_name = 9;
    int64 port_address = 10;
    CommandResponse cmd_response = 11;
    }
    Parameter Description
    instance_id New instance for the device in the enumeration
    device_alias Unique identifier (alphanumeric) for every device in the enumeration
    device_type Indicates the type of device like Motion stage, Safety Controller etc (defined below in the DeviceType enum)
    device_model_name Indicate the model name for each device (define below in the DeviceModel enum)
    device_status Indicate whether the device is online , offline or not reachable (define below in the device status enum)
    ipaddr Indicate the ipaddress of the device
    mac_addr Indicate the mac address of the device
    Conn_handle Handle object to keep the device open
    port_name Port name
    port_address Port address
    cmd_response Indicates command response that describes API response code and its description (Defined below in CommandResponse)

    enum DeviceType{
    NONE = 0;
    SENSOR_3D = 1;
    MOTION_STAGE = 2;
    CAMERA = 3;
    IO_MODULE = 4;

    message CommandResponse{
    int32 resp_code = 1;

    }
    ParameterDescription
    resp_code Response code from the device API
    resp_message Response message

GLOSSARY

  • 3D Laser Profiler : A device used for measuring surface profiles and shapes in three dimensions using laser technology.
  • LJ-V Series : A series of 3D laser profilers manufactured by Keyence, including models like LJ-V7000, LJ-V7300, LJ-V7500, etc.
  • Profile : A cross-sectional view of the surface of an object captured by the 3D laser profiler, showing height or depth information.
  • Measurement Range : The distance over which the 3D laser profiler can accurately measure surfaces.
  • Resolution : The smallest change in height or depth that the 3D laser profiler can detect.
  • Repeatability : The ability of the 3D laser profiler to produce consistent results when measuring the same object multiple times.
  • Sampling Interval : The distance between individual measurement points along the profile.
  • Point Cloud : A set of data points in three-dimensional space representing the surface of an object, captured by the 3D laser profiler.
  • Calibration : The process of adjusting the settings of the 3D laser profiler to ensure accurate measurements.
  • Alignment : The process of positioning the 3D laser profiler relative to the object being measured to obtain the desired measurement results.
  • Ethernet/IP : A communication protocol used for connecting the 3D laser profiler to other devices or systems in an industrial automation environment.
  • Measurement Tools : Software features of the 3D laser profiler used for analyzing profiles, such as height measurements, area calculations, and defect detection.
  • Stable Run : A mode in which the 3D laser profiler continuously monitors and adjusts its settings to maintain stable and accurate measurements.
  • Trigger Mode : A mode in which the 3D laser profiler captures a profile only when triggered by an external signal, allowing for precise control of measurement timing.
  • Surface Flatness : A measurement of how flat or smooth a surface is, typically expressed as a deviation from a perfectly flat plane.
  • Height Deviation : The difference in height between a point on the surface and a reference plane, used to assess surface roughness or profile irregularities.
  • Dynamic Range : The range of surface heights that the 3D laser profiler can measure, from the lowest to the highest point.
  • Deflection : The bending or deformation of an object's surface, measured by comparing its profile to a reference profile.
